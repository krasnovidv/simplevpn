# Jenkins unable to find Jenkinsfile — file not committed to git

**Date:** 2026-03-09
**Files:** Jenkinsfile
**Severity:** medium

## Problem

Jenkins pipeline fails immediately with: "Unable to find Jenkinsfile from git file:///opt/simplevpn-src". The build never starts.

## Root Cause

The Jenkinsfile was created locally but never added to git (listed as `??` untracked). Jenkins clones/checks out the repo and looks for Jenkinsfile in the repo root — since it's not tracked, it doesn't exist in the checkout.

Secondary issue: the repo URL `file:///opt/simplevpn-src` is a Unix-style path, but Jenkins runs on Windows with `bat` steps. This path won't resolve on Windows.

## Solution

Committed the Jenkinsfile to git so Jenkins can find it during checkout. The repo URL in Jenkins job config needs manual update to the correct Windows path (e.g., `file:///D:/Projects/rknpnh`).

## Prevention

- Always verify new CI config files are committed before running the pipeline
- When setting up Jenkins jobs, double-check the repo URL matches the OS where Jenkins runs
- Consider using a remote git URL instead of local file paths for CI
